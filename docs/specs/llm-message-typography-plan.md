## План: унификация типографики ответов LLM

Цель: добиться одинакового визуального межстрочного и межабзацного расстояния
в ответах агента независимо от того, пришел ли чистый текст или markdown.

Plan approved: no

### Контекст (что именно ломает единый вид)
- LLM-ответы рендерятся через `MessageResponse` (Streamdown/markdown),
  разные markdown-структуры дают разные отступы/line-height.
- Reasoning рендерится отдельно через `ReasoningContent` (Streamdown).
- Fallback/прочие виды сообщений используют обычный `<p>` с `leading-relaxed`.
- В итоге даже при одном и том же компоненте `Message` визуальный результат разный.

### Спецификации, которые нужно проверить/обновить
- `docs/specs/agents/requirements.md` — добавить/уточнить требование про единый
  внешний вид сообщений агента (межстрочное/межабзацное расстояние) независимо
  от формата ответа (text/markdown).
- `docs/specs/agents/design.md` — описать единый набор правил типографики для
  LLM-ответов и reasoning; обновить примеры/таблицу покрытия при необходимости.
- `docs/specs/llm-integration/requirements.md` — проверить, нужно ли уточнение
  в разделе UI отображения (если решим закрепить требование там).
- `docs/specs/llm-integration/design.md` — проверить, нужно ли доп. описание
  единого markdown-рендера/типографики (если влияет на дизайн).

### План действий
1) Зафиксировать целевой визуальный стандарт:
   - line-height для основного текста LLM,
   - межабзацные отступы,
   - spacing для списков/код-блоков,
   - поведение для одиночных переносов строк.
2) Реализация на уровне рендера markdown:
   - задать единый класс/набор правил для `MessageResponse` (Streamdown),
   - при необходимости задать одинаковые стили для `ReasoningContent`,
   - убедиться, что `MessageContent`/контейнер не добавляет лишние отступы.
3) Прогнать визуальные сценарии вручную:
   - чистый текст (1 абзац),
   - несколько абзацев,
   - список,
   - код-блок,
   - mixed markdown.
4) Обновить спецификации (см. раздел выше).
5) Тестирование:
   - после значимых изменений запускать `npm run test:unit`,
   - по завершении — `npm run validate`,
   - спросить про запуск функциональных тестов (они покажут окна).
