# Документ Требований: Window Management

## Введение

Данный документ описывает требования к управлению главным окном приложения Clerkly, включая конфигурацию окна при запуске, поведение, визуальное оформление и сохранение состояния.

## Глоссарий

- **Main Window** - Главное окно приложения
- **Maximized State** - Развернутое состояние окна (занимает весь экран, но не полноэкранный режим)
- **Fullscreen Mode** - Полноэкранный режим (скрывает системные элементы macOS)
- **Title Bar** - Заголовок окна приложения
- **Window State** - Состояние окна (позиция, размер, развернутое состояние)
- **Window Bounds** - Границы окна (x, y, width, height)
- **Database** - Локальная база данных SQLite, управляемая через DatabaseManager
- **workAreaSize** - Размер рабочей области экрана (весь экран минус системные элементы macOS)
- **WindowManager** - Компонент для управления жизненным циклом главного окна
- **WindowStateManager** - Компонент для управления персистентностью состояния окна (использует глобальное хранилище, не изолированное по пользователям)
- **DatabaseManager** - Единая точка входа для доступа к базе данных SQLite (см. user-data-isolation.6)

## Архитектурный Принцип

Приложение следует архитектурному принципу **Единого Источника Истины (Single Source of Truth)**, описанному в спецификации `clerkly` (см. `docs/specs/clerkly/requirements.md`).

**Применение к Window Management:**
- Состояние окна (позиция, размер, развернутость) сохраняется в базе данных
- При изменении состояния окна данные обновляются в базе
- При запуске приложения состояние восстанавливается из базы данных

## Требования

### 1. Конфигурация Главного Окна при Запуске

**ID:** window-management.1

**User Story:** Как пользователь, я хочу чтобы приложение запускалось в компактном окне, оптимизированном для чатов с агентами, чтобы интерфейс был сфокусирован и не занимал весь экран.

**Зависимости:** Нет

#### Критерии Приемки

1.1. КОГДА приложение запускается впервые (нет сохраненного состояния), ТО "Main Window" ДОЛЖНО открываться с размером min(900, ширина экрана) x min(800, высота экрана), но НЕ в maximized состоянии

1.2. "Main Window" ДОЛЖНО НЕ открываться в полноэкранном режиме (fullscreen)

1.3. "Main Window" ДОЛЖНО позволять пользователю изменять размер окна (resizable: true)

1.4. "Main Window" ДОЛЖНО позволять пользователю максимизировать окно через стандартные элементы управления macOS

1.5. "Main Window" ДОЛЖНО сохранять видимость системных элементов macOS (меню, dock)

1.6. "Main Window" ДОЛЖНО иметь минимальную ширину 350px и минимальную высоту 650px, и пользователь НЕ ДОЛЖЕН иметь возможность уменьшить окно меньше этого размера

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should open at default size on first launch"
- `tests/functional/window-state-persistence.spec.ts` - "should open application with correct initial window state"

### 2. Заголовок Главного Окна

**ID:** window-management.2

**User Story:** Как пользователь, я хочу чтобы окно приложения имело чистый интерфейс без лишнего текста в заголовке, чтобы интерфейс выглядел минималистично.

**Зависимости:** Нет

#### Критерии Приемки

2.1. КОГДА приложение запускается, ТО "Main Window" ДОЛЖНО иметь пустой заголовок (title: '')

2.2. "Main Window" ДОЛЖНО НЕ отображать название приложения в заголовке окна

2.3. "Main Window" ДОЛЖНО сохранять стандартные элементы управления окном macOS (кнопки закрыть, свернуть, развернуть)

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should have empty window title"

### 3. Нативный Mac OS X Интерфейс

**ID:** window-management.3

**User Story:** Как пользователь Mac OS X, я хочу чтобы приложение использовало нативные элементы интерфейса и следовало стандартным конвенциям macOS, чтобы приложение ощущалось как родное для моей системы.

**Зависимости:** Нет

#### Критерии Приемки

3.1. "Main Window" ДОЛЖНО использовать нативные Mac OS X элементы управления окном

3.2. "Main Window" ДОЛЖНО использовать стандартные Mac OS X элементы интерфейса (кнопки, меню, диалоги)

3.3. "Main Window" ДОЛЖНО следовать стандартным конвенциям активации приложения

3.4. "Main Window" ДОЛЖНО интегрироваться с системным dock macOS

### 4. Адаптация Размера Окна к Экрану

**ID:** window-management.4

**User Story:** Как пользователь, я хочу чтобы приложение адаптировалось к размеру моего экрана, чтобы окно корректно отображалось на любом мониторе.

**Зависимости:** window-management.1

#### Критерии Приемки

4.1. КОГДА приложение запускается впервые, ТО "Main Window" ДОЛЖНО определять размер экрана пользователя

4.2. КОГДА сохраненное состояние отсутствует (первый запуск), ТО "Main Window" ДОЛЖНО использовать размер по умолчанию: min(900, ширина экрана) x min(800, высота экрана)

4.3. КОГДА сохраненное состояние существует, ТО "Main Window" ДОЛЖНО использовать сохраненные размеры из предыдущей сессии

4.4. "Main Window" ДОЛЖНО центрировать окно на экране при первом запуске

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should open at default size on first launch"
- `tests/functional/window-state-persistence.spec.ts` - "should adapt window size to small screens"

### 5. Сохранение Состояния Окна

**ID:** window-management.5

**User Story:** Как пользователь, я хочу чтобы приложение запоминало позицию и размер окна, чтобы при следующем запуске окно открывалось в том же месте и с тем же размером.

**Зависимости:** window-management.1, window-management.4

#### Критерии Приемки

5.1. КОГДА пользователь изменяет размер окна, ТО "Main Window" ДОЛЖНО сохранять новый размер в постоянное хранилище

5.2. КОГДА пользователь перемещает окно, ТО "Main Window" ДОЛЖНО сохранять новую позицию в постоянное хранилище

5.3. КОГДА пользователь разворачивает или сворачивает окно, ТО "Main Window" ДОЛЖНО сохранять состояние развернутости

5.4. КОГДА приложение запускается повторно, ТО "Main Window" ДОЛЖНО восстанавливать сохраненные размер, позицию и состояние (включая maximized состояние, если окно было закрыто максимизированным)

5.5. КОГДА сохраненное состояние отсутствует (первый запуск), ТО "Main Window" ДОЛЖНО использовать поведение по умолчанию (размер min(900, ширина экрана) x min(800, высота экрана), центрированное на экране, но не в maximized состоянии)

5.6. КОГДА сохраненная позиция находится за пределами доступных экранов, ТО "Main Window" ДОЛЖНО открываться на основном экране

5.7. Состояние окна ДОЛЖНО быть общим для всех пользователей (НЕ изолировано по пользователям)

**Примечание:** В отличие от других данных приложения (настройки, профиль, токены), состояние окна является глобальным и не зависит от авторизованного пользователя. Это обеспечивает консистентное поведение окна независимо от того, какой пользователь вошел в систему.

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should persist window size across restarts"
- `tests/functional/window-state-persistence.spec.ts` - "should persist window state across restarts"
- `tests/functional/window-state-persistence.spec.ts` - "should persist maximized state across restarts"

### 6. Завершение Приложения при Закрытии Окна

**ID:** window-management.6

**User Story:** Как пользователь, я хочу чтобы приложение завершалось при закрытии главного окна, чтобы не оставлять фоновые процессы работающими после того, как я закрыл окно.

**Зависимости:** Нет

#### Критерии Приемки

6.1. КОГДА пользователь закрывает главное окно, ТО приложение ДОЛЖНО полностью завершиться

6.2. КОГДА все окна закрыты, ТО приложение ДОЛЖНО завершить работу независимо от платформы (macOS, Windows, Linux)

6.3. КОГДА приложение завершается, ТО все данные ДОЛЖНЫ быть корректно сохранены перед выходом

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should quit application when main window is closed"

#### Нефункциональные Требования

**Производительность:**
- Загрузка сохраненного состояния окна должна выполняться менее чем за 50ms
- Сохранение состояния окна должно выполняться асинхронно и не блокировать UI
- Окно должно появляться на экране менее чем за 500ms после запуска приложения

**Надежность:**
- Система должна корректно обрабатывать поврежденные данные состояния окна
- При ошибках загрузки/сохранения состояния приложение должно продолжать работу
- Состояние окна должно сохраняться даже при аварийном завершении приложения (через события close)

**Совместимость:**
- Поддержка различных конфигураций мониторов (один монитор, несколько мониторов)
- Корректная работа при изменении конфигурации мониторов между запусками
- Адаптация к различным разрешениям экрана (от 800x600 до 4K и выше)

### 7. Открытие Внешних Ссылок в Браузере по Умолчанию

**ID:** window-management.7

**User Story:** Как пользователь, я хочу чтобы при клике на ссылку в сообщениях агента она открывалась в системном браузере по умолчанию, чтобы удобно переходить по внешним ресурсам.

**Зависимости:** Нет

#### Критерии Приемки

7.1. КОГДА пользователь кликает на внешнюю ссылку (http:, https:, mailto:) в контенте приложения (в т.ч. в сообщениях агента), ТО ссылка ДОЛЖНА открываться в системном браузере по умолчанию

7.2. Открытие ссылки НЕ ДОЛЖНО создавать новое окно приложения или встроенный браузер

7.3. Переход внутри приложения (внутренняя навигация) НЕ ДОЛЖЕН перехватываться

#### Функциональные Тесты

- `tests/functional/llm-chat.spec.ts` — тесты рендеринга markdown links (косвенно)

## Вне Области Применения

Следующие элементы явно исключены из данной спецификации:

- Кастомизация внешнего вида заголовка окна (цвет, стиль)
- Поддержка других операционных систем (Windows, Linux)
- Сохранение состояния содержимого окна (только геометрия окна)
- Управление несколькими окнами приложения
- Полноэкранный режим (fullscreen mode)

## Технические Ограничения

- **Платформа**: Только macOS (использование нативных элементов управления macOS)
- **Electron API**: Использование BrowserWindow и screen API
- **База данных**: SQLite через DatabaseManager (глобальное хранилище для состояния окна)
- **Формат хранения**: JSON для сериализации состояния окна
- **Размер окна по умолчанию**: min(900, ширина экрана) x min(800, высота экрана) при первом запуске
- **Минимальный размер окна**: ширина >= 350px, высота >= 650px
- **Максимальный размер окна**: Ограничен размером экрана (workAreaSize)
