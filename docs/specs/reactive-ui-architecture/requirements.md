# Документ Требований: Reactive UI Architecture

## Введение

Данный документ описывает кросс-фичевые требования к реактивной архитектуре UI в приложении Clerkly: как renderer-компоненты подписываются на snapshot-события и обновляют состояние без дублирования бизнес-логики.

## Фокус Документа

Документ фиксирует только кросс-фичевый UI data-flow слой:
- реактивные подписки UI на доменные snapshot-события;
- правила применения снапшотов в hooks/store/component state;
- запрет дублирования бизнес-вычислений в UI при наличии вычисленных полей в snapshot.

Документ НЕ определяет:
- transport/IPC/event typing и delivery semantics (см. `docs/specs/realtime-events/*`);
- правила рендера конкретных экранов/компонентов (см. профильные UI-спеки, в первую очередь `docs/specs/agents/*`);
- LLM/tool-loop/runtime оркестрацию (см. `docs/specs/llm-integration/*`, `docs/specs/code_exec/*`).

## Глоссарий

- **Reactive UI Subscription** — подписка UI-компонента/хука на доменные события со snapshot payload
- **Snapshot** — полное актуальное состояние сущности на момент события
- **UI Data-Flow** — путь обновления UI: событие → hook/store → компонент

## Требования

### 1. Реактивные подписки UI

**ID:** reactive-ui-architecture.1

**User Story:** Как разработчик, я хочу чтобы UI обновлялся реактивно по snapshot-событиям, чтобы исключить дополнительный polling и лишние запросы.

#### Критерии Приемки

1.1. UI-компоненты ДОЛЖНЫ получать актуальные данные через реактивные подписки на snapshot-события.

1.2. КОГДА снапшот содержит вычисляемое поле (например, статус), ТО UI SHALL использовать это поле напрямую и SHALL NOT пересчитывать его повторно в renderer.

1.3. КОГДА приходит snapshot-событие создания/обновления/архивации сущности, ТО соответствующий UI state ДОЛЖЕН обновляться без дополнительных запросов к main process для той же сущности.

1.4. Подписки ДОЛЖНЫ ограничиваться только релевантными сущностями/наборами сущностей (single-entity и list-уровень).

1.5. КОГДА компонент/хук прекращает жизненный цикл, ТО подписки ДОЛЖНЫ корректно очищаться.
