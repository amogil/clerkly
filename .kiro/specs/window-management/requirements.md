# Документ Требований: Window Management

## Введение

Данный документ описывает требования к управлению главным окном приложения Clerkly, включая конфигурацию окна при запуске, поведение, визуальное оформление и сохранение состояния.

## Глоссарий

- **Main Window** - Главное окно приложения
- **Maximized State** - Развернутое состояние окна (занимает весь экран, но не полноэкранный режим)
- **Fullscreen Mode** - Полноэкранный режим (скрывает системные элементы macOS)
- **Title Bar** - Заголовок окна приложения
- **Window State** - Состояние окна (позиция, размер, развернутое состояние)
- **Window Bounds** - Границы окна (x, y, width, height)
- **Database** - Локальная база данных SQLite, управляемая через DataManager
- **workAreaSize** - Размер рабочей области экрана (весь экран минус системные элементы macOS)

## Архитектурный Принцип

Приложение следует архитектурному принципу **Единого Источника Истины (Single Source of Truth)**, описанному в спецификации `clerkly` (см. `.kiro/specs/clerkly/requirements.md`).

**Применение к Window Management:**
- Состояние окна (позиция, размер, развернутость) сохраняется в базе данных
- При изменении состояния окна данные обновляются в базе
- При запуске приложения состояние восстанавливается из базы данных

## Требования

### 1. Конфигурация Главного Окна при Запуске

**ID:** window-management.1

**User Story:** Как пользователь, я хочу чтобы приложение запускалось в окне, занимающем весь доступный экран, чтобы максимально использовать пространство для работы, и иметь возможность изменять его размер и максимизировать.

**Зависимости:** Нет

#### Критерии Приемки

1.1. КОГДА приложение запускается впервые (нет сохраненного состояния), ТО "Main Window" ДОЛЖНО открываться размером с workAreaSize (весь экран минус системные элементы macOS), но НЕ в maximized состоянии

1.2. "Main Window" ДОЛЖНО НЕ открываться в полноэкранном режиме (fullscreen)

1.3. "Main Window" ДОЛЖНО позволять пользователю изменять размер окна (resizable: true)

1.4. "Main Window" ДОЛЖНО позволять пользователю максимизировать окно через стандартные элементы управления macOS

1.5. "Main Window" ДОЛЖНО сохранять видимость системных элементов macOS (меню, dock)

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should open at default size on first launch"

### 2. Заголовок Главного Окна

**ID:** window-management.2

**User Story:** Как пользователь, я хочу чтобы окно приложения имело чистый интерфейс без лишнего текста в заголовке, чтобы интерфейс выглядел минималистично.

**Зависимости:** Нет

#### Критерии Приемки

2.1. КОГДА приложение запускается, ТО "Main Window" ДОЛЖНО иметь пустой заголовок (title: '')

2.2. "Main Window" ДОЛЖНО НЕ отображать название приложения в заголовке окна

2.3. "Main Window" ДОЛЖНО сохранять стандартные элементы управления окном macOS (кнопки закрыть, свернуть, развернуть)

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should have empty window title"

### 3. Нативный Mac OS X Интерфейс

**ID:** window-management.3

**User Story:** Как пользователь Mac OS X, я хочу чтобы приложение использовало нативные элементы интерфейса и следовало стандартным конвенциям macOS, чтобы приложение ощущалось как родное для моей системы.

**Зависимости:** Нет

#### Критерии Приемки

3.1. "Main Window" ДОЛЖНО использовать нативные Mac OS X элементы управления окном

3.2. "Main Window" ДОЛЖНО использовать стандартные Mac OS X элементы интерфейса (кнопки, меню, диалоги)

3.3. "Main Window" ДОЛЖНО следовать Mac OS X конвенциям поведения окна при закрытии

3.4. "Main Window" ДОЛЖНО следовать Mac OS X конвенциям активации приложения

3.5. "Main Window" ДОЛЖНО интегрироваться с системным dock macOS

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should have native Mac OS X window controls"
- `tests/functional/window-state-persistence.spec.ts` - "should follow Mac OS X window close conventions"
- `tests/functional/window-state-persistence.spec.ts` - "should integrate with Mac OS X dock"

### 4. Адаптация Размера Окна к Экрану

**ID:** window-management.4

**User Story:** Как пользователь, я хочу чтобы приложение адаптировалось к размеру моего экрана, чтобы окно корректно отображалось на любом мониторе.

**Зависимости:** window-management.1

#### Критерии Приемки

4.1. КОГДА приложение запускается впервые, ТО "Main Window" ДОЛЖНО определять размер экрана пользователя

4.2. "Main Window" ДОЛЖНО использовать размеры экрана для определения начальных размеров окна

4.3. "Main Window" ДОЛЖНО НЕ использовать хардкоженные размеры (например, 1920x1080)

4.4. КОГДА экран пользователя меньше стандартного размера, ТО "Main Window" ДОЛЖНО адаптировать размеры к доступному пространству

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should open at default size on first launch"

### 5. Сохранение Состояния Окна

**ID:** window-management.5

**User Story:** Как пользователь, я хочу чтобы приложение запоминало позицию и размер окна, чтобы при следующем запуске окно открывалось в том же месте и с тем же размером.

**Зависимости:** window-management.1, window-management.4

#### Критерии Приемки

5.1. КОГДА пользователь изменяет размер окна, ТО "Main Window" ДОЛЖНО сохранять новый размер в постоянное хранилище

5.2. КОГДА пользователь перемещает окно, ТО "Main Window" ДОЛЖНО сохранять новую позицию в постоянное хранилище

5.3. КОГДА пользователь разворачивает или сворачивает окно, ТО "Main Window" ДОЛЖНО сохранять состояние развернутости

5.4. КОГДА приложение запускается повторно, ТО "Main Window" ДОЛЖНО восстанавливать сохраненные размер, позицию и состояние

5.5. КОГДА сохраненное состояние отсутствует (первый запуск), ТО "Main Window" ДОЛЖНО использовать поведение по умолчанию (окно размером с workAreaSize, но не в maximized состоянии)

5.6. КОГДА сохраненная позиция находится за пределами доступных экранов, ТО "Main Window" ДОЛЖНО открываться на основном экране

#### Функциональные Тесты

- `tests/functional/window-state-persistence.spec.ts` - "should persist window size across restarts"

## Вне Области Применения

Следующие элементы явно исключены из данной спецификации:

- Кастомизация внешнего вида заголовка окна (цвет, стиль)
- Поддержка других операционных систем (Windows, Linux)
- Сохранение состояния содержимого окна (только геометрия окна)
- Управление несколькими окнами приложения
- Полноэкранный режим (fullscreen mode)
